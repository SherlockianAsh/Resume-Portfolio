GitHub Pages — Custom Domain Setup

Repo: SherlockianAsh/Resume-Portfolio  |  Domain: ashlock-tech.my  |  Deploy: GitHub Actions

1 Configure GitHub Pages Source

  1. Go to Repository Settings → Pages
  2. Under Build and deployment, set Source to GitHub Actions
  3. The workflow (.github/workflows/deploy.yml) is already in the repo — it will build and deploy on every push to main
Important Do NOT select "Deploy from a branch" — the repo uses GitHub Actions with actions/deploy-pages.

2 Set Custom Domain in GitHub

  1. On the same Pages settings page, find Custom domain
  2. Enter: ashlock-tech.my
  3. Click Save
  4. GitHub will show a DNS check — it will fail until Step 3 is done
CNAME file A public/CNAME file containing ashlock-tech.my is already committed in the repo. This tells GitHub Pages to serve under your custom domain even after redeployment.

3 Configure DNS Records

Log in to your domain registrar (where you bought ashlock-tech.my) and add DNS records.

Option A: Apex domain (ashlock-tech.my)

Add four A records pointing to GitHub's IP addresses:

TypeHost / NameValueTTL
A@185.199.108.1533600
A@185.199.109.1533600
A@185.199.110.1533600
A@185.199.111.1533600

Also add a www CNAME so www.ashlock-tech.my works too:

TypeHost / NameValueTTL
CNAMEwwwSherlockianAsh.github.io3600

Option B: Subdomain only (e.g. resume.ashlock-tech.my)

If you prefer a subdomain instead:

TypeHost / NameValueTTL
CNAMEresumeSherlockianAsh.github.io3600
If using a subdomain Update the custom domain in GitHub Pages settings to match (e.g. resume.ashlock-tech.my), and update public/CNAME in the repo to the same value.

4 Verify DNS Propagation

DNS changes can take 5 minutes to 48 hours to propagate. Verify with:

# Check A records
dig ashlock-tech.my +short

# Expected output (all four IPs):
# 185.199.108.153
# 185.199.109.153
# 185.199.110.153
# 185.199.111.153

# Check CNAME for www
dig www.ashlock-tech.my +short

# Expected: SherlockianAsh.github.io.

Or use an online tool: whatsmydns.net

On Windows (PowerShell) Use Resolve-DnsName ashlock-tech.my instead of dig.

5 Enable HTTPS

  1. Go back to Repository Settings → Pages
  2. Wait until the DNS check shows a green checkmark
  3. Check Enforce HTTPS
  4. GitHub will auto-provision a Let's Encrypt SSL certificate (takes a few minutes)
Cannot enable HTTPS? If the checkbox is greyed out, DNS hasn't propagated yet. Wait and refresh. GitHub needs to verify domain ownership before issuing the SSL cert.

6 Verify Everything Works

  • Visit https://ashlock-tech.my — landing page loads
  • Visit https://ashlock-tech.my/resume — resume page loads
  • Visit https://ashlock-tech.my/login — login page loads
  • Refresh on /resume — SPA redirect works (no 404)
  • Browser shows padlock icon (HTTPS active)
  • https://www.ashlock-tech.my redirects to apex domain
